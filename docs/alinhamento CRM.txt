Objetivo da Integração

Integrar o CRM da Fundação com o Tasy para que o processo comercial aconteça no CRM e o Tasy fique responsável pela operação e geração dos contratos.

Fluxo Esperado
Venda realizada no CRM.
CRM envia os dados para o Tasy via JSON.
Tasy:
Consulta se o contratante já existe (CNPJ/CPF).
Atualiza cadastro se existir.
Cadastra se não existir.
Tasy cria a proposta de adesão.
Tasy processa beneficiários e portabilidades.
Tasy gera o contrato.
Assinatura eletrônica ocorre através do fluxo do Tasy.
Após conclusão:
É gerado o número da carteirinha.
Retorna informação para o CRM.
Sobre os JSONs

Pelo que foi dito:

Já existem 3 JSONs definidos.
Esses JSONs representam os dados necessários para integração.
A equipe do Tasy deverá desenvolver APIs para receber esses JSONs.
O CRM consumirá/executará essas APIs para enviar os dados.

Cenário 1 — Kotlin como camada de integração (mais comum)

O CRM envia:

{
  "cnpj": "12345678000199",
  "razaoSocial": "Empresa XYZ",
  "beneficiarios": [...]
}

A API Kotlin recebe:

@PostMapping("/propostas")
fun criarProposta(
    @RequestBody request: PropostaRequest
): ResponseEntity<Any> {

    val resultado = tasyService.criarProposta(request)

    return ResponseEntity.ok(resultado)
}

O Service chama o Tasy:

@Service
class TasyService(
    private val tasyClient: TasyClient
) {

    fun criarProposta(request: PropostaRequest): PropostaResponse {
        return tasyClient.enviarProposta(request)
    }
}

E o Client chama uma API do Tasy:

@FeignClient(name = "tasy")
interface TasyClient {

    @PostMapping("/api/propostas")
    fun enviarProposta(
        @RequestBody request: PropostaRequest
    ): PropostaResponse
}
